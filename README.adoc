* *Hugo* v0.151.2 - Générateur de sites statiques
* *GitHub Actions* - CI/CD
* *Markdown* & *AsciiDoc* - Format de contenu

== Contribution

Workflow de contribution :

. Cloner le dépôt
   ↓
. Créer une branche
   ↓
. Créer/Modifier le contenu
   ↓
. Commiter les changements
   ↓
. Pousser et créer une Pull Request
   ↓
. Review par l'équipe
   ↓
. Merge et déploiement automatique

=== Cloner le dépôt

[source,bash]
----
cd existing_repo
git remote add origin https://src.koda.cnrs.fr/gdr-scilog/private/site.git
git branch -M main
git push -uf origin main
----

WARNING: Il est fort probable que vous deviez renseigner votre token d'accès GitHub/GitLab pour pouvoir cloner le dépôt. L'URL du site devra alors être https://votre.login.de.connexion@src.koda.cnrs.fr/gdr-scilog/private/site.git

=== Créer une branche 

[source,bash]
----
git checkout -b feature/ajout-my-first-post
----

Convention de nommage :

`feature/` : Nouveau contenu
`fix/` : Correction
`update/` : Mise à jour

=== Créer/Modifier un contenu

Soit en utilisant l'outil Hugo si vous avez l'habitude :

[source,bash]
----
hugo new content content/uncategorized/my-first-post.md
----

L'avantage d'utiliser Hugo est de pouvoir tester en local :
[source,bash]
----
hugo server -D
----

=== Commiter

[source,bash]
----
git add content/uncategorized/my-first-post.md
git commit -m "feat(post): ajout mon premier post"
----

Convention de commit :

type(scope): description

Types :
- `feat`: nouveau contenu
- `fix`: correction
- `docs`: documentation
- `style`: mise en forme
- `refactor`: restructuration
- `chore`: maintenance

=== Pousser et créer une PR

[source,bash]
----
git push origin feature/ajout-my-first-post
----

=== Déploiement

Après review, tests automatiques (CI) et validation → Merge.    
Déploiement automatique via GitHub Actions.
Site mis à jour sous quelques minutes.
